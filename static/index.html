Ce que le bot analyse

Trois domaines couverts

🔒

Sécurité

Mots de passe en dur, données sensibles exposées et injections SQL.

⚡

Performance

Boucles inutiles, mauvaises complexités et optimisations possibles.

✦

Propreté

Code dupliqué, noms peu clairs et fonctions trop longues.

Comment ça fonctionne

Entièrement automatique

01

Tu ouvres une Pull Request

GitHub détecte l'événement automatiquement.

02

GitHub notifie le bot

Une notification est envoyée au serveur déployé sur Railway.

03

Le code modifié est récupéré

Les lignes changées sont extraites via l'API GitHub.

04

L'IA analyse le code

Groq / LLaMA 3.3 identifie les problèmes et propose des corrections.

05

Le commentaire est posté

L'analyse apparaît directement sur ta Pull Request.
